# ✅ Task A Checklist – Analysing the Problem (2026)

## 1. Understand the Client Brief

☐ Read the brief **carefully at least twice**
☐ Identify the **main problem the client wants solved**
☐ Identify the **goal of the system/product**
☐ Highlight any **important constraints** (time, budget, technology)

---

## 2. Identify Stakeholders

☐ List **primary stakeholders** (e.g., client, users)
☐ List **secondary stakeholders** (e.g., staff, administrators, customers)
☐ Explain **how each stakeholder will use the system**
☐ Identify **their needs and expectations**

---

## 3. Define the Problem Clearly

☐ Write a **clear problem statement**
☐ Explain **why the problem exists**
☐ Describe **what happens if the problem isn’t solved**
☐ Identify **current issues with the existing system/process**

---

## 4. Gather Requirements

☐ List **Functional Requirements** (what the system must do)
☐ List **Non-Functional Requirements** (performance, security, usability, etc.)
☐ Identify **user requirements**
☐ Identify **system requirements**

Examples:

* login system
* search function
* secure data storage
* fast loading times

---

## 5. Identify Constraints

☐ Technical constraints (software/hardware)
☐ Time constraints (deadline)
☐ Legal constraints (e.g., **UK General Data Protection Regulation / data protection)
☐ Budget constraints

---

## 6. Identify Risks

☐ List possible **project risks**
☐ Explain **how each risk could affect the project**
☐ Suggest **ways to reduce the risk**

Examples:

* data breach
* system crashes
* users not understanding the system

---

## 7. Research Similar Solutions

☐ Research **existing systems or websites**
☐ Identify **what they do well**
☐ Identify **their weaknesses**
☐ Explain **how your solution could improve them**

---

## 8. Define Success Criteria

☐ List **clear success criteria** for the final product
☐ Make sure they are **measurable**
☐ Link them to the **client requirements**

Example:

* Users can create an account
* Data is stored securely
* Pages load within 3 seconds

---

## 9. Summarise Your Analysis

☐ Write a **short conclusion** summarising:

* the problem
* the key requirements
* the main stakeholders
* the overall goal of the project

---

✅ **Tip for higher marks:**
Explain **why** each requirement is important, not just **what** it is.

Example:
Instead of writing

> “The system must have login.”

Write

> “The system must include a login feature to ensure only authorised users can access sensitive data, improving security and meeting data protection requirements.”
